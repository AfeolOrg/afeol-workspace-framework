# Contributing to AFEOL Workspace Framework

Thank you for your interest in contributing.

The AFEOL Workspace Framework is a specification-first project. Long-term stability, clarity, and architectural consistency take precedence over rapid feature growth.

---

# Before Contributing

Please read the following documents before proposing changes:

1. README.md
2. Workspace Foundation
3. Relevant derived specifications

Architectural discussions should always be based on the existing specifications.

---

# Guiding Principles

Every contribution should preserve the following principles:

- Technology neutrality
- Separation of responsibilities
- Deterministic behavior
- Security by design
- Long-term compatibility
- Clear and explicit specifications

Implementation convenience shall never override a Foundation rule.

---

# Normative Changes

Changes affecting normative requirements should:

- clearly explain the motivation;
- preserve compatibility whenever reasonably possible;
- avoid unnecessary complexity;
- maintain consistency with existing specifications.

Breaking changes should be introduced only when they provide substantial architectural benefit.

---

# Pull Requests

A good pull request should:

- address one topic;
- include clear reasoning;
- avoid unrelated modifications;
- keep documentation synchronized with specification changes.

Large architectural redesigns should be discussed before implementation.

---

# Coding Style

Reference implementations should favor:

- readability;
- maintainability;
- simplicity;
- explicit behavior;
- deterministic execution.

Programming language choices are left to individual implementations unless restricted by a specific specification.

---

# Documentation

Documentation is considered part of the implementation.

Changes to normative behavior should always include corresponding documentation updates.

---

# Questions and Discussions

Questions, proposals, and architectural discussions are welcome.

Constructive technical discussion is encouraged.

Personal attacks, disrespectful behavior, and non-technical arguments are not acceptable.

---

Thank you for helping improve the AFEOL Workspace Framework.
