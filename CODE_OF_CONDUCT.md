# Code of Conduct

## Our Commitment

The AFEOL Workspace Framework aims to provide a respectful, professional, and technically focused environment for everyone participating in the project.

All contributors, maintainers, reviewers, and participants are expected to act with professionalism and mutual respect.

---

## Expected Behavior

Participants are expected to:

- communicate respectfully;
- focus discussions on technical topics;
- provide constructive feedback;
- accept technical disagreement professionally;
- support an open and collaborative environment.

---

## Unacceptable Behavior

The following behavior is not acceptable:

- personal attacks;
- harassment or discrimination;
- intimidation or threats;
- abusive language;
- deliberately disruptive behavior;
- intentionally misleading technical information;
- spam or unrelated promotional content.

---

## Project Philosophy

Architectural decisions should be based on technical merit, evidence, long-term maintainability, and consistency with the published specifications.

Disagreement is expected.

Disrespect is not.

---

## Scope

This Code of Conduct applies to:

- GitHub Issues;
- Pull Requests;
- Discussions;
- project documentation;
- official communication channels related to the project.

---

## Enforcement

Project maintainers are responsible for interpreting and enforcing this Code of Conduct.

They may edit, remove, lock, or reject contributions that violate these guidelines.

Repeated or severe violations may result in temporary or permanent exclusion from participation.

---

## Reporting

If you believe this Code of Conduct has been violated, please contact the project maintainers through the official GitHub repository.

Reports will be reviewed fairly and handled as confidentially as reasonably possible.

---

Thank you for helping build a professional and welcoming open-source project.
